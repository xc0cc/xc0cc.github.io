Anaconda与Jupyter

Anaconda 与 Jupyter：数据科学的环境管理入门

初学 Python 做数据分析，第一个坑往往不是代码，是环境——装了几个库版本冲突、系统 Python 被搞乱、项目 A 要 Python 3.9 项目 B 要 3.11。Anaconda 就是解决这些的：提供一套虚拟环境管理系统，每个项目跑在隔离环境里，互不干扰。Jupyter 则是写数据分析代码最顺手的工具——浏览器里跑代码，分段执行，边写边看结果。

Anaconda

它是什么

Anaconda 是一个数据科学平台，核心包含三部分：

  • Python 发行版——预装 NumPy、Pandas、Scikit-learn、TensorFlow 等常用科学计算库
  • Conda 包管理器——安装、更新、管理 Python 包和环境
  • 虚拟环境系统——创建多个互相隔离的 Python 运行环境

简单说：可以同时有一套 Python 3.9 + TensorFlow 2.x 的环境，和一套 Python 3.12 + PyTorch 的环境，各自独立运行，像容器一样互不干扰。

Anaconda、Miniconda、Conda 的区别

这三个名字经常混着用，实际上是不同的东西：

是什么 包含什么
Conda 包管理工具 只有包管理器本身
Miniconda 精简发行版 Python + Conda + 基础依赖
Anaconda 完整发行版 Python + Conda + 几百个预装库 + 图形界面

推荐装 Miniconda。 Anaconda 预装了几百个库，大部分用不上，占用十几个 G。Miniconda 只带 Python 和 Conda，需要什么库自己装，体积小得多。

官网：https://www.anaconda.com/

Jupyter

它是什么

Jupyter 是一个基于浏览器的交互式开发工具，三个特点：

  • 代码块分段执行——不是从头跑到底，逐个 cell 运行，调试高效
  • 内嵌 Markdown——代码中间插入说明文字、数学公式、图表，适合边分析边记录
  • 多语言支持——主要写 Python，也支持 R、Julia 等

Jupyter 包含两个产品：Jupyter Notebook（经典单文档界面）和 JupyterLab（多文档、终端、文件浏览器集成，更像 IDE）。日常使用 Notebook 就够了，Lab 功能更全但稍重。

官网：https://jupyter.org/

总结

Anaconda 管环境，Jupyter 管编写。配合起来的标准流程：创建项目专用环境 → 装依赖 → 在 Jupyter 里选对应环境 → 分段写代码跑分析。比”全局装一堆库互相打架 → 写脚本 → 跑 → 报错”的体验好太多。
