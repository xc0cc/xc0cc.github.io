服务器

基本概念

内部结构及其概念

2U2路 其中U代表高度 1U=4.45厘米
路代表cpu数量

CPU 提供计算资源

硬盘 存放永久数据，保存应用和运算后的文件（当开启应用时，再读取硬盘数据道内存中运行），容量较大，效率相比于内容低。

风扇 为CPU和内部元件提供散热功能

导风罩 导风，让服务器能够集中散热

散热器 为CPU提供散热功能

内存条 存放CPU运算结果（存放”易失性数据”），容量较少，效率相比硬盘更高

主板 提供插槽，连接计算、存储、I/O 等资源。也是服务器最重要的资源，一旦损坏需要整机更换

RAID卡 为服务器提供硬盘控制功能（如：RAID 1、RAID 3 等不同的捆绑方式，提供主备服务功能）

接口 (网口、USB、VGA 等接口)提供内外部设备的 I/O 接入功能

GPU 提供图形和图像的计算资源（服务器中不常见，一般为外置的。个人终端则集成在CPU 上或者主板上）

RAID

分为软件、硬件

RAID0

数据条带化，就是从逻辑上看出一块硬盘，速度快，容量大，但是没有冗余，一块坏了，数据火葬场

RAID1

一比一镜像，写入1X，读取2X速度。

RAID3

数据条带话，但是异或校验，有备份，最少三个硬盘，2X+1X，数据恢复时，进行异或校验。

RAID5

数据条带化、“斜列式”异或校验、有备份，最少三块硬盘，每一块盘及时存储盘，也是校验盘。

RAID6

（N+2 块硬盘组成）（N 必须大于等于 2 ）最少 4 块硬盘组成
工作方式：P+Q 分布式校验来进行计算（2 对分布式校验算法来做出备份
DP 其中两块是校验盘（一块是横向校验盘、一块是斜列式校验盘，同时组成数据备份）

RAID10

先做RAID1，在做RAID0。

存储类型

文件系统，数据存入硬盘，文件系统会安装某种特定的文件格式和大小空间存入
安装文件系统，需要格式化，也就是整个硬盘写0

保险箱盘：用于保存存储设备的配置信息、日志信息、用户配置和系统掉电后 Cache 中的脏数据

内置保险箱盘：主要为盘控一体的设备
外置保险箱盘：盘控分离的设备使用

Cashe

镜像Cache

一般双活的设备，都会有对方的写入数据，通过镜像 Cache 实现，实现冗余。
例如：A 收到用户的写请求，写入数据 Data 1，控制器 A 会把缓存在 Cache 中的数据复制一份，通过镜像链路传递给控制器 B 的镜像 Cache
（B同理） 作用：避免控制器故障，用户数据丢失

写Cache

由于硬盘写入数量受到接口 I/O 速率的限制，有可能在大量数据写入时，无法及时写入硬盘
这部分数据称为“脏数据”写 Cache 可以先缓存这部分数据，等待硬盘空闲时继续写入

脏数据：当一个事务正在访问数据，并且对数据进行了修改，而这种修改还没有提交到数据库中或者磁盘中。

回写：当用户把数据写入 Cache 中时，控制器会立刻回复写反馈（告知用户已经完成写入动作）
达到阀值才会刷盘才会统一写入

透写：当用户把数据写入 Cache 中，控制器会根据硬盘当前的速率把用户数据写入到硬盘

​ （对于用户而言，大容量的数据写入反馈较慢。但是写入速度相对来说是最，工作周期快）

读Cache

由于硬盘的读取速率也会受到接口的 I/O 速率影响，所以在读取数据时可以使用读 Cache 来提升读取速率

智能预取：会根据用户对文件的访问情况，来提前预取部分数据，例如读到ABC时，DE写入Cashe

固定预取：根据用户指定的大小来对连续文件进行预取，固定读取，读1，写2，读3，写4

倍数预取：按照文件读取的大小来进行倍数预取，适合于大小不一的随机读取文件

集中存储

SAN

大概就是一大堆硬盘做RAID然后给服务器使用

LUN 是 SAN 存储中，其中一部分的磁盘集集合后（RAID 后）提供的存储空间（条带）
可以理解为多个硬盘同一位置的一部分集合，这部分集合的存储空间称为 LUN
（LUN 是存储设备，用于给计算服务器提供存储空间的，相当于服务器内部的一个本地磁盘）
由于 LUN 本身是不带文件系统的，所以服务器收到后，需要先进行格式化（安装文件系统）才能使用

NAS

和san的区别就是文件系统在NAS这边，而且就像一台比较弱的电脑，需要通过网络去使用。

分布式存储

相比于（SAN、NAS）等存储有其他高级的虚拟化特性（快照、链接克隆等）

虚拟化

背景 为了在不造成资源冲突的前提下，提供资源的利用率

优点

  1. 提高资源利用率
  2. 使资源相对独立，互不抢占
  3. （提高了资源的利用率，相比原有服务器来说，可以降低能耗和节能）
  4. 操作系统和硬件实现解耦

特点

  1. 分区：通过虚拟化层把物理资源进行逻辑划分
  2. 隔离：通过分区把物理资源逻辑划分，每一份资源都是独享的，并且相互隔离
  3. 封装：虚拟机的配置可以像文件一样被封装起来
  4. 相对硬件独立：虚拟机于底层物理硬件没有紧耦合关系，相对硬件和主机操作系统来说，虚拟机就是一个应用层序

分类

Ⅰ型虚拟化（裸金属虚拟化）

直接在物理主机上，安装虚拟化层（虚拟化操作系统）通过虚拟化层对物理资源进行池化
优点：虚拟化的利用率高，速度更快（虚拟机的性能接近原生操作系统）
缺点：开发难度大，相当于重新设计了一个主机操作系统，需要对主机操作系统代码进行修改
目前使用的 KVM 就是基于 Linux 的（Windows 和 Mac OS 都是闭源的）

Ⅱ型虚拟化（宿主虚拟化 / 寄居虚拟化）

在已经安装了操作系统的主机上，部署虚拟化层（如：VMware）该虚拟化层相对于主机操作系统而言
只是一个应用程序，由该应用程序提供虚拟化功能
优点：无需对主机操作系统内核进行修改，可以支持原生操作系统
缺点：虚拟化效率较低，损耗大

补充

为了保护 CPU，指令被分为 4 个等级（Ring 0 —— Ring 3）

Ring 0 对应的是内核态：内核态可以执行特权指令（也可以执行普通指令，Ring 0 可以执行 1-3 所有的指令）

Ring 3 对应的是用户态：用户态可以执行普通指令

特权指令：是指对计算机核心资源进行访问的指令（包括，分配物理硬件资源的指令、清除内存、修改时钟频率等）

非特权指令（用户指令）：是指应用程序下发的普通指令，例如 QQ 发消息、传递文件等

系统调用：当执行特权指令的时候，主机操作系统会切换到内核态下执行，当完成特权操作后又会切换为用户态

（由于 Guest OS 认为自己运行在真实的服务器上，并且能够调用服务器所有的资源，所以和 Host OS 一样，拥有 Ring 0 等级）
可以切换到内核态去执行特权指令，但是在一个计算节点上，可能会拥有多台虚拟机，这些虚拟机都有自己的操作系统
虚拟机之间就会导致资源抢占问题以及共享物理资源是容易导致系统错误，为了避免这个情况，需要对 Guest OS 进行降级

特权解除：把 Guest OS 所处的等级进行降级处理，降为 Ring 1（解除 Guest OS 的特权）
为了避免 VM 之间进行资源的抢占问题

特权解除后，虚拟机 Guest OS 下发的特权指令，都为敏感指令
一般情况下，这些敏感指令都由 Hypervisor 去模拟执行

全虚拟化 对于虚拟机下发的敏感指令，全都会由 Hypervisor 去进行捕抓，进行二进制翻译（陷入模拟）
无论翻译后的指令是否特权指令都由 Hypervisor 交给物理服务器处理（KVM 支持全虚拟化）

优点：全虚拟化不需要修改虚拟机操作系统，只需要 VMM 对 Guest OS 下发的指令进行捕抓即可

缺点：所有的敏感指令都需要陷入模拟，进行翻译处理，需要占用大量的 CPU 资源, 导致虚拟化效率较低

半虚拟化 通过修改 Guest OS 的源码，让 Guest OS 意识到工作在虚拟机上，并且让其识别这 19 条非特权的敏感指令
当 Guest OS 收到敏感指令时，如果属于这 19 条指令之一就会直接通过 Ring 1 等级处理
如果发现该敏感指令属于特权指令，则会通过超级调用交给 Hypervisor 并且切换到内核态（Ring 0 等级）处理
（XEN 和 KVM 都支持半虚拟化）

优点：虚拟化层则提供了与原始系统相近的性能，因为该虚拟化层其实就是主机操作系统,并且 Guest OS 通过识别敏感指令减少了陷入模拟的操作，使得虚拟化效率更高

缺点：半虚拟化中的 Host OS 只有针对开源的系统才能支持被修改，如 Linux，而对于未开源的，如：Windows 系统，则无法实现半虚拟化技术

硬件辅助虚拟化

通过物理硬件 CPU 来对不同的指令进行识别
Guest OS 不再进行特权解除，但 Guest OS 下发的指令，CPU 不会采用内核态去执行。会把 Guest OS 下发的指令处于非 Root 模式下执行
如果是特权指令则切换到 Root 模式下执行，解决了虚拟机资源调用冲突问题

优点：不需要和全虚拟化、半虚拟化一样进行陷入模拟，进一步提升效率

缺点：厂商的物理硬件需要对虚拟化支持（目前主流的 X86 芯片都支持硬件辅助虚拟化）

XEN

特权虚拟机： （Domian 0）能够直接通过驱动软件来访问物理资源
非特权虚拟机：（Domain U）不能直接访问物理资源，需要先通过前端驱动找到特权虚拟机
“下发请求指令通过前端驱动找到特权虚拟机，特权虚拟机收到请求后授权处理这些指令”,再通过特权虚拟机的驱动软件调用资源\

优点：相比 KVM 更安全，虚拟机之间无法之间访问物理资源，需要授权处理

缺点：多台虚拟机同时调度时，受 I/O 接口限制，资源访问率慢（虚拟化效率底）

KVM

物理资源由内核中的 KVM 模块进行虚拟化，KVM 模拟会把虚拟资源映射给上层设备使用（VM）
每台 VM 都各自有自己的操作系统（Guest OS）并且拥有 QUME 模块
当虚拟机需要获取物理资源时，Guest OS 会直接下发敏感指令，QUME 模块捕抓后进行二进制翻译，并且通过内核模块
交给主机操作系统模拟处理，主机操作系统可以通过驱动软件访问物理资源

优点：虚拟化性能接近原生操作系统

缺点：安全性较差，所有的虚拟机共享操作系统内核。

​ UVP：对服务器资源进行虚拟化，是一个虚拟化软件
​ VNA：管理虚拟化资源（可以对虚拟化资源进行调用和配置）

​ 可以对 CNA 主机进行统一管理

​ 通过该平台分配资源等

CPU虚拟化

上面的虚拟化部分

内存虚拟化

内存大页：支持 2M 的内存页，内存页越大所需要描述的内存页表就越少，寻找内存地址的时候就越快

MMU：把虚拟内存地址映射给物理内存地址的工具（例如：使用 HVA 地址，需要把地址中的映射和索引关系，找到物理内存地址的具体位置）
如果使用物理内存地址之前，每一次都需要这样寻址，会很耗费计算资源，所以就使用 TLB 来进行记录

TLB：缓存虚拟内存地址和物理内存地址映射关系的表格。当应用使用物理内存时，就先通过 TLB 中寻找对应记录，如果有则直接映射
如果没有（未击中 TLB）则会通过 MMU 重新进行映射

影子页表：用于记录 GVA 的映射关系以及 HPA 的映射关系，使得虚拟机可以通过影子页表来找到物理主机的 HPA（GVA > HPA）
大大减少了 MMU 的查找时间，提升性能
优点：提升了内存虚拟化的转换效率，提高了新能
缺点：影子页表占用大量的内存空间（存放于内存的 Cache 中）因为这个问题，影子页表也在淘汰的边缘

EPT（扩展页表）：是由 Inter 支持的技术，可以将 GVA > GPA > HPA 的关系进行内存转换。实现硬件支持内存虚拟化的功能

超大内存页：内存大页可以支持 2M — 1G 的内存页，1G 的是超大内存页。在特定场景下可以使用（大规格的虚拟机，以 T 的内存单位的设备）

内存复用

多个虚拟机映射同一块内存空间，对该空间只有访问权限。（一般是未写入数据的剩余空间，填 0 操作）

如果需要写入新的数据，则需要从物理内存中映射其他新的空间

内存置换

把长时间没有访问的内存数据，写入到硬盘中，并释放这部分内存空间

当需要访问这个数据的时候，又从硬盘重新提取出来并加载到内存中

内存气泡

例如有2台虚拟机，VM1 和 VM2 （VM1 占用的内存较多，VM2 占用的内存较少）内存气泡，会把 VM2 的空闲内存通过气泡

I/O模拟

实现虚拟机共享有限的 I/O 资源，减少冲突问题

全虚拟化：通过 QEMU 模块完全模拟物理网卡的功能，提高给虚拟机（让虚拟机认为自己可以通过驱动来实现 I/O 访问）
性能最差，但是兼容性好，不需要额外购买 I/O 资源

半虚拟化：通过虚拟机安装特殊的驱动（virtio）来实现对后端的交互，实现访问物理网卡的功能
性能次之，但是需要特殊的驱动（有可能导致无法兼容，而无法实现 I/O 虚拟化）
I/O 独享：直接通过 VT-d 来把 PCIe 插槽映射给虚拟机使用，在通过插入物理网卡来支持虚拟机的访问
使得虚拟机使用网卡时，与物理主机使用网卡一样
性能最好，不需要进行虚拟化（独占一个 PCIe 插槽，主板插槽有限。外置 I/O 设备需要额外购买，成本较高）

HCIP

FusionSphere

单虚拟化场景 只使用 FusionCompute 来提服务器虚拟化，的应用场景。满足企业日常业务需求

多虚拟化场景 使用（FusionCompute + vphere） 来提供服务器虚拟化功能，并结合 FusionManager 来进行纳管的场景

私有云场景 （FusionCompute + FusionAccess + VPC）方案来实现私有云应用场景

集群管理

虚拟机 HA（高可靠性）

作用：当虚拟机所在的主机故障时，虚拟机能够迁移到其他主机上快速拉起
减少故障中断的时间，避免业务无法继续使用

虚拟机热迁移

作用：在不影响用户使用的情况下，进行虚拟机迁移
可以结合例如（DRS、DPM 来减轻设备的负担，和绿色节能）

  1. 迁移之前，CNA 主机之间必须能访问共享存储（读取虚拟机磁盘文件的数据）
  2. CNA-1 的 VM 迁移到 CNA-2 上使用
  3. CNA-2 开启一台虚拟机（VM-2）虚拟机拥有同等的配置参数
    CNA-2 会读取共享存储中的虚拟机磁盘文件
  4. 然后 CNA-1 把 VM-1 寄存器和内存中的数据，进行快照定格（保存里面数据信息）
    然后再传递给 CNA-2，并且中通发生变更的数据也会重新进行快照定格，通过 N 个内存变更分片进行传递
  5. FusionCompute 会对 VM-1 进行 I/O 悬挂，等待最后的内存变更分片传递完成后，切断用户与 VM-1 的联系
    修改至 VM-2 上，并且取消 I/O 悬挂，完成迁移

DRS（动态资源调度）

作用：实现 FusionCompute 平台中的集群 CNA 能够进行负载均衡，削峰填谷（减少 CNA 主机之间负载失衡的问题）

DPM（分布式电源管理）

作用：根据用户的使用程度，来进行虚拟机的迁移（与 DRS 类似，可以对虚拟机进行调度）

IMC

作用：可以把虚拟机迁移到不同的 CNA 主机上工作（CNA 主机的 CPU 代数可以不一样，要是要兼容高>低）

存储

存储资源

能够提供存储空间的物理服务器（例如：IP SAN 存储服务器）华为的：FusionStorage（分布式存储）、OceanStor， 存储资源不能直接给 CNA 主机去使用，需要进行存储空间的管理和划分。

  1. 创建硬盘域（把硬盘集合在一起）
  2. 从硬盘域中通过 RAID 划分出不同容量大小的存储池
  3. 从存储池中，划分出物理的存储管理空间（例如：LUN、共享文件、FusionStorage 存储池）

存储设备

从存储池中划分出来的物理存储管理单元，或者是本地磁盘
第一种：通过挂载在 CAN 上，形成虚拟存储资源，虚拟化存储
第二种：通过直接映射挂载给虚拟机使用，裸设备映射

数据存储

是指把存储设备交给 FusionCompute 进行虚拟化管理（先把存储设备映射成逻辑存储管理单元）
然后再把这些逻辑存储管理的单元，按照需求划分给虚拟机使用（中间会经过虚拟化层）

虚拟化存储

可以提供虚拟化高级特性（快照、链接克隆、存储热迁移、存储扩容等）
存储资源通过虚拟化层来模拟使用，会有性能损耗（ 性能再 3种 存储中时最差的）

裸设备映射

会丢失存储高级特性（不支持：快照、链接克隆、存储热迁移等）
虚拟机之间使用物理存储设备，不存在虚拟化层调度（性能是最好的）

非虚拟化存储

通过 FusionStorage 把存储设备映射给虚拟机使用，不需要经过 CNA 本身的虚拟化层
保留了部分高级特性（支持快照、存储热迁移、扩容等功能）（性能次之）

VIMS （FusionCompute 中的文件管理系统）：可以实现多个主机节点，能同时访问共享的存储空间

**DLM ** （分布式锁机制）：只允许 VIMS 中的 Master 设备对共享存储空间进行读写操作
Slave 只有读的权限，避免了数据不一致性问题
主机之间，通过 VIMS 来建立连接（VIMS 主机之间的心跳线）当 Master 故障后，其他的设备会重选 Master 并且获取读写权限
主机于共享存储之间也会建立磁盘心跳线，来检测主机和存储设备之间的网络连通性

格式化方式

普通置零：虚拟机获取的磁盘空间，与申请的磁盘空间一致（立刻下发）

精简磁盘：虚拟机获取磁盘空间时，最初始的时候空间为 0（包含磁盘的元数据，大小在几十K 到几M之间）

普通延时置零：虚拟机在获取磁盘空间时，与申请的磁盘空间一致（立刻下发）需要时再写0

差分盘：基于源数据盘来进行创建，用于记录源数据盘中的差异数据或者增改数据。可以用于（快照、连接克隆等）

差分盘 附属于母盘，（默认）差分盘不能脱离于母盘使用，可以用于保存（快照数据、链接克隆、作为独立-非持久盘的增量空间使用）

快照

快照：可以记录虚拟机当前的状态，并通过快照文件保存（虚拟机的寄存器、内存数据、虚拟机的配置等）

全量快照：类似于 RAID 1 ，把所有原盘文件进行镜像复制
创建速度慢，占用空间大（快照恢复时间短）
读写速度快，不需要同时读或者同时写入

差分快照：（相比于全量快照，创建速度快、占用空间较小）
适合于对 IOPS 要求低，但是空间节省，利用率高

ROW（写时重定向）：差分盘对于母盘重新定向读写（新的数据写入到差分盘，母盘改为只读）

COW（写时复制）：差分盘只会保留母盘变更的数据（新数据写入母盘时，差分盘会对母盘源数据进行写复制）

从属盘：从属于母盘的存储空间（母盘+从属盘）容量不能大于虚拟机所分配的磁盘空间

独立-持久盘：类似于 U盘 ，独立的存储空间。写入和删除数据都是永久存储在的

独立-非持久盘：类似于系统的还原卡，与独立盘特性一样，快照不会对独立-非持久盘进行快照保留

存储热迁移过程

  1. 虚拟机热迁移前，虚拟机对原磁盘文件拥有完整的读写权限
  2. 开始存储热迁移时，虚拟机会通过原磁盘文件（母盘）在目的数据存储中创建出与之对应的差分盘。差分盘通过指针方式来描述原磁盘文件中的数据，使得虚拟机通过差分盘也能获取到母盘数据（此时，虚拟机不再对母盘拥有写入权限，只存在读取权限）
  3. 当母盘数据开始迁移时，VIMS 文件系统会通过 TAR（分块压缩传输）技术来对 ROW 快照做逆操作
    （原本 ROW 的母盘修改数据时，新的数据会写入到差分盘中。母盘需要恢复时，删除差分盘的数据进行回滚即可，也就是以母盘为主）
    （逆操作的时候则以差分盘为主，母盘数据写入差分盘中）
  4. 虚拟机获取差分磁盘的 I/O 权限（读写）并且把新的数据写入差分盘中，此时母盘也会使用 TAR 技术把数据迁移到 ROW 的差分盘中，当母盘数据迁移完成后。目的端的差分磁盘就获得了所有的数据（即：全量数据）
    但是，差分盘不能独立于母盘使用，所以删除母盘前需要将差分盘修改为动态磁盘类型（即：把差分盘修改为精简磁盘或普通延迟置零磁盘）这时候目的磁盘才能独立使用
